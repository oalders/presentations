git config --global \
alias.revert-file-in-commit \
'!f() { git show $2 -- $1 | git apply -R;}; f'
  • Revert the changes introduced by a single file in a single commit.
  • Invoke as "git revert-file-in-commit path/to/file commit"
  • which translates to: 
    git show dedca812826 -- path/to/file | git apply -R

How is this generally useful?

  • Use ! to run an external command via git alias
  • Use an anonymous function for complex logic
  • This can make it easier to deal with pipes
  • You can easily accept multiple arguments
